# Security Policy

## Supported Versions

Only the latest `v2.x` release receives security patches.

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Use GitHub Private Vulnerability Reporting to disclose issues responsibly:

**[Report a vulnerability](https://github.com/arielespinoza07/result-pattern/security/advisories/new)**

### What to include

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

### Response time

| Stage           | Target        |
| --------------- | ------------- |
| Acknowledgement | Within 72 h   |
| Patch release   | Within 14 days (severity-dependent) |

You will receive a confirmation once your report is received and again when a fix is released.
Thank you for helping keep this project secure.
